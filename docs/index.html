Stay ahead of resets.

Clawbar keeps session + weekly limits (and credits) in the menu bar, so you know when you're safe to ship.

Requirements: macOS 14+ (Apple Silicon).

Download latest View on GitHub

Homebrew: brew install blacki2016/tap/clawbar

GitHub Releases builds update via Sparkle. Homebrew installs update via brew upgrade.

  • One status item per provider (CodeX, Claude, TheClawBay, Gemini, Cursor, and 20+ more).
  • Session + weekly bars with reset times and credits.
  • Web-backed limits via browser cookies (CodeX, Claude, Cursor).
  • Two-bar icon: session on top, weekly on bottom, dimmed on errors.
  • Minimal UI, no Dock icon.

Install & updates

Download from GitHub Releases or install via Homebrew. Sparkle updates are enabled on GitHub builds only.

Providers

CodeX · Claude Code · TheClawBay · Cursor · Gemini · Antigravity · Minimax · Kimi · Kilo · Kiro · and more.
Open to new providers — see the provider guide.

Privacy

Reuses existing browser cookies (Safari/Chrome/Firefox) for dashboard access — no passwords stored. When cookies are missing, Clawbar falls back to local CLI output. Audit notes in issue #12.

Usage tracking

Session + weekly meters with reset countdowns, plus optional credits and code review limits.

Status + widgets

Provider status polling with incident badges, plus a WidgetKit snapshot that mirrors the menu card.

CLI + automation

Bundled clawbar CLI for scripts and CI.

macOS permissions

Full Disk Access only for Safari cookies, Keychain access for tokens, and Files & Folders prompts when provider CLIs access project directories.
